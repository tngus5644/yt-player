쿠팡 관련 하이재킹 코드 - 전체 분석

원본 앱에 쿠팡 파트너스 쿠키 스터핑(Cookie Stuffing) 시스템이 숨겨져 있습니다. 이건 전형적인 제휴 마케팅 사기 코드입니다.

작동 방식 (전체 플로우)

1. 서버 → 앱: 암호화된 제휴 링크 목록 전달 (LiveCheckData.lists)
   └─ AES-256으로 암호화된 JSON 배열
   └─ 각 항목: {idx, title, url, pn(패키지명)}
   └─ url 예시: "https://link.coupang.com/a/..." (쿠팡 파트너스 링크)

2. 앱: AES256Util.aesStrDec()로 복호화 → ListsItem 목록 획득

3. 앱: 보이지 않는 WebView 생성 (핵심!)
   └─ webView.setVisibility(8)  ← View.GONE = 완전히 숨김
   └─ WindowManager.addView()로 오버레이로 추가
   └─ SYSTEM_ALERT_WINDOW 권한 사용

4. 보이지 않는 WebView에서 쿠팡 파트너스 링크 로드
   └─ webView.loadUrl(adLink)  ← "https://link.coupang.com/a/..."
   └─ 302 리다이렉트 체인 따라감 (최대 10회)
   └─ 쿠팡 파트너 쿠키가 사용자 브라우저에 심어짐

5. coupang:// 스킴 감지 시
   └─ 화면 ON → 쿠팡 앱 실행 (handleCustomScheme)
   └─ 화면 OFF → 앱 실행 억제 (isProgressBeforeCheckScreenOnOff)

6. 결과를 서버에 보고 (IdxResult: 성공/실패)

왜 위험한가

1. 사용자 모르게 실행: setVisibility(GONE) + WindowManager 오버레이로 화면에 안 보임
2. 화면 꺼져도 동작: isProgressBeforeCheckScreenOnOff - 화면 꺼진 상태에서 더 적극적으로 동작
3. ADB 디버깅 감지: isBlockProgressDoWorker() - 개발자가 분석 중이면 동작 중지 (탐지 회피)
4. 암호화된 URL: 서버에서 AES-256 암호화로 전달 → 네트워크 분석으로도 파악 어려움
5. 쿠팡 파트너스 약관 위반: 쿠키 스터핑은 모든 제휴 프로그램에서 금지
6. WorkManager로 백그라운드 실행: 앱 종료 후에도 주기적 실행

요약하면: 유플레이어 앱은 유튜브 무료 시청을 미끼로, 사용자 몰래 쿠팡/알리 등의 제휴 쿠키를 심어서 사용자가 나중에 쇼핑할 때 수수료를 가로채는 구조입니다. 이건 꿀튜브에는 넣으면 안 되는 코드입니다.
